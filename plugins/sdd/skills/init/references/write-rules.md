# BUILD-Phase Write Rules

Rules for Phase 2 of `/init` — writing `feature-spec/config.yaml`. Load this file when SCAN is confirmed and BUILD is about to start. `config.yaml` is the **only** file `/init` writes.

---

## Write `feature-spec/config.yaml`

Read the template at `skills/init/templates/config.yaml` and fill it in:

- `tech_stack:` — the one-line stack string from detection § 1. Single source of truth for versions.
- `architecture:`
  - `pattern:` — detected/confirmed architecture pattern (string).
  - `layers:` — `name → path` pointers, never file lists.
  - `entry_points:` — `kind → path` for every entry location detected (HTTP, pages, middleware, plugins, jobs, event handlers, CLI), or the repo's non-standard entry convention.
  - `hard_rules:` — structural invariants only (one-line imperatives). Omit lint-enforceable and historical/version-pinned rules.
- `lint_commands:` — detected lint/format commands (fix mode). Omit if none.
- `verification_commands:` — detected type-check / unit-test / build commands. Omit if none.
- `rules:` — populate only if project-specific artifact rules were detected; otherwise omit.
- Update the header comment date: `# Auto-generated by /init on YYYY-MM-DD`.

### Content guidelines

- **Pointer-over-enumeration**: `layers` and `entry_points` use `name → path`. If you find yourself listing the files under a path, stop — point at the directory with a cardinality instead.
- **Versions live only in `tech_stack`**: strip version numbers from `architecture`, `hard_rules`, and `rules` before writing.
- **`hard_rules` are invariants linters cannot express** (layer boundaries, allowed call directions, cross-component contracts). Anything an existing lint config enforces is dropped.
- **Empty is valid**: leave a field as `""` / `[]` when SCAN marked it ○ or found nothing confidently. Do not invent placeholder content.
- **Comments stay**: keep the template's explanatory comments for the optional sections (`lint_commands` / `verification_commands` / `rules`) so a human editing the file later has guidance.
